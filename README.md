🎮 《後驗紀錄》目前已實現之核心功能 (Current Features)
本專案目前已將遊戲的「輸入法底層」與「AI 神經網絡」完全打通，以下為目前開發進度與功能說明：

1. 🔠 進階微軟/Mac 輸入法鍵盤監聽系統
為了在 Pygame 環境中完美相容中文輸入，專案整合了 TEXTINPUT 與自訂事件監聽（事件 770 與 771）：

動態組字預覽：玩家輸入注音符號時，尚未按確認選字的拼音會以 亮黃色加底線 即時渲染在輸入框中。

正式文字安全移入：按下空格選字確認（或輸入英文）時，文字會轉為 青色 正式移入輸入緩衝，並自動清空組字緩衝，徹底解決英文重複輸入、疊字與衝突問題。

退格功能：支援完整的 Backspace 退格鍵，可安全刪除已確認輸入的字元。

字數與發送安全防護：單次輸入上限為 30 個字。當 AI 正在思考或仍有未確認的注音符號時，按下 Enter 會安全阻斷，避免誤送空字串或閃退。

2. 🧠 多線程異步 AI 對話架構 (最新 Google GenAI SDK)
最新官方套件整合：全面捨棄舊版廢棄套件，改用 Google 2025/2026 最新官方推薦的 google.genai（genai.Client 與 chats.create）架構，建立具備上下文記憶力的持續對話模式。

非阻塞異步線程 (Threading)：利用 threading.Thread 將 API 網路請求移至背景執行。玩家送出文字後，遊戲畫面維持流暢不卡死，並在 UI 上即時呈現 （皮歐里正在沉思中...） 的動態灰色提示。

內容安全解鎖：內建自訂的 SafetySetting，將所有敏感與危險內容的攔截機制全面設為 BLOCK_NONE，確保核心黑闇、創傷劇本發揮不受限。

連線異常防護：內建 try-except 錯誤攔截，若網路斷線或 API 異常，會自動輸出符合人設的警戒對白（皮歐里沉默不語，似乎在警戒著什麼...），遊戲絕不崩潰。

3. 📊 動態遊戲數據連動 (Game Stats Integration)
引擎物件精準對齊：對話系統已與主引擎 GameEngine 與 game_state 完整對接。

實時數值打包發送：每次打字送出時，系統會自動抓取主引擎當前的 皮歐里好感度 (favorability)、精神污染度 (corruption)、總部猜疑度 (suspicion)、玩家善惡值 (karma)、故事章節 (chapter) 以及 Numen 真相探索度 (numen_truth)，作為 Prompt 的上下文同步發送給 Gemini，確保 AI 的回覆完全符合遊戲當前數值現狀。

系統提示詞 (System Instruction) 綁定：皮歐里（42歲、前 C.E.A. 成員、矛盾心境）的設定與遊戲核心規則已深植在 AI 的意識設定中。

4. 📈 關鍵字動態信任演算
好感度即時加減：即時監聽玩家文字。偵測到「幫助、謝謝、拜託、對不起」等善意詞彙，自動幫 time_manager.favorability +10；偵測到「欺騙、不信任、懷疑、假」等詞彙則自動 -10。

動態章節世界線切換：好感度 > 90 自動跳轉至第三章，> 70 進入第二章，其餘維持第一章，同步動態改變發送給 AI 的場景狀況描述。

線索進度解鎖：當玩家輸入文字包含「numen」（不限大小寫）時，自動觸發線索進度，幫 numen_truth +10。

5. 🖼️ 對話 UI 渲染與基礎劇本播放
雙模式切換：支援固定劇情對白播放模式（start_normal_dialogue，按 Space/Enter 跳下一句，播完關閉）與 AI 自由互動模式（start_ai_dialogue）。

日常結算機制：主引擎具備獨立練習的經驗值結算與日常行動邏輯基礎。

# 遺漏部分
1. 結局判定系統 (Ending Conditions)
這部分需要與你的四大數值（好感度、精神污染度、總部猜疑度、善惡值）以及動態章節進行聯動：

數值臨界點監聽：當玩家與皮歐里對話送出、或是進行日常行動後，需要檢查數值是否觸發特定結局。例如：suspicion（總部猜疑度）達到 100% 觸發「總部清洗結局」；corruption（精神污染度）過高觸發「失控狂暴結局」。

多線結局分歧：目前對話系統已經可以根據 favorability（好感度）動態切換第 1 至第 3 章，後續需要實作在第 3 章結束時，依據 karma（善惡值）與 numen_truth（Numen 真相探索度）決定最終走向（是殺死皮歐里、被皮歐里除掉，還是共同揭開 C.E.A. 的陰謀）。

2. 食物與生存系統 (Food & Survival Mechanics)
作為在「404號據點」封閉空間生活的設定，物資管理是營造氛圍的關鍵：

物資消耗邏輯：需要實作每日結束（或每次行動）後，食物與飲水的消耗。

數值聯動：缺乏食物時，應對應扣除玩家的體力，甚至間接導致 corruption（精神污染度）加速上升。

餵食/共享事件：與皮歐里分享食物，或是由皮歐里主動提供食物時，應觸發特殊的動態對話，並獲得好感度加成。

3. 皮歐里運行軌跡與 AI 行為修正 (NPC Pathfinding & Behavior)
修正目前地圖上「奇怪的運行軌跡」，讓 NPC 的行動更符合邏輯與人設：

狀態機切換（FSM）：根據遊戲內的時間或玩家狀態，切換皮歐里的行為。例如：玩家獨自練習時，皮歐里必須處於「不在場」或「巡邏中」的狀態，否則會抓到玩家偷練非治療魔法。

地圖碰撞與路徑修正：檢查動態路徑演算法，避免皮歐里在 404 據點內卡在牆壁、家具，或是出現瞬間移動、無意義來回走動的詭異軌跡。

4. 美術渲染與 UI 優化 (Visual Rendering & Juice)
有了流暢的輸入法，視覺體驗也需要同步跟上：

對話框精緻化：目前是基礎的 pygame.draw.rect 線條框。後續可以替換成帶有科技感、或破損斑駁感的美術貼圖（UI Sprite）。

打字機效果 (Typewriter Effect)：NPC 的回覆文字與劇情對話，應實作逐字顯示的打字機效果，而不是一瞬間整行跳出來。

頭像與表情差分：根據 AI 回傳的氛圍（或是根據好感度、章節狀態），在對話框旁動態渲染皮歐里的冷酷、警戒、或偶爾展現同情的表情頭像。

5. 施放魔法與技能系統 (Magic Casting System)
對應人設與核心規則的魔法訓練機制：

技能練習限制：實作當皮歐里在場（陪同訓練）時，玩家只能選擇「治療魔法」；若強行施放或練習「攻擊」、「開鎖」，需要立刻扣除 favorability 並觸發皮歐里的警戒對話。

獨自練習機制：當皮歐里移出當前地圖（自己練習）時，解鎖攻擊與開鎖的練習功能，獲取對應的經驗值（如你畫面上顯示的 【練習成功】），且此時不影響皮歐里的數值。
